## üå± Shallow Copy (Definition)

* A **shallow copy** of an array creates a **new array**, but the elements inside are **referenced**, not deeply copied.
* If elements are **primitives** (numbers, strings, booleans) ‚Üí changing the copy does not affect the original.
* If elements are **objects or arrays** ‚Üí changes to inner objects **will reflect** in the original.

---

## ‚úÖ Examples

### 1Ô∏è‚É£ Using `slice()`

```js
const arr = [1, 2, 3];
const copy = arr.slice();

copy[0] = 100;

console.log(arr);  // [1, 2, 3]
console.log(copy); // [100, 2, 3]
```

* Works fine for primitives ‚Üí **changes in copy do not affect original**.

---

### 2Ô∏è‚É£ Using Spread Operator (`...`)

```js
const arr = [1, 2, 3];
const copy = [...arr];

copy[1] = 200;

console.log(arr);  // [1, 2, 3]
console.log(copy); // [1, 200, 3]
```

---

### 3Ô∏è‚É£ Shallow Copy with Objects inside

```js
const arr = [{a: 1}, {b: 2}];
const copy = [...arr];

copy[0].a = 100;

console.log(arr);  // [{a: 100}, {b: 2}]
console.log(copy); // [{a: 100}, {b: 2}]
```

* **Why?** Because the inner objects are **still referenced**.
* That‚Äôs why it‚Äôs called **shallow** copy, not deep copy.

---

## ‚ö° Summary

* Shallow copy ‚Üí new array, but **references to nested objects remain the same**.
* Methods to create shallow copy:

  1. `slice()` ‚Üí `arr.slice()`
  2. Spread ‚Üí `[...arr]`
  3. `Array.from()` ‚Üí `Array.from(arr)`

---

### **1Ô∏è‚É£ Spread Operator (ES6+)**

```javascript
const newArray = [...array];
```

* Creates a **shallow copy**.
* Easy and readable.

---

### **2Ô∏è‚É£ `slice()` Method**

```javascript

```

* Returns a **shallow copy** of the array.
* Works in **all modern browsers**.

---

### **3Ô∏è‚É£ `Array.from()`**

```javascript

```

* Creates a new array from an existing one.
* Shallow copy.

---

### **4Ô∏è‚É£ `concat()` Method**

```javascript

```

* Concatenates with an empty array ‚Üí returns a **new array**.

## üå± Deep Copy (Definition)

*A **deep copy** creates a new array and also copies all nested objects/arrays recursively.

Changes in the deep copy do not affect the original array, even for nested objects.


### **5Ô∏è‚É£ For Objects or Nested Arrays (Deep Copy)**

* The above methods only make **shallow copies**.
* For nested arrays/objects, use:

```javascript
const newArray = JSON.parse(JSON.stringify(array));
```

* Makes a **deep copy**, so inner arrays/objects are also copied.

---

‚úÖ **Summary:**

* **Shallow copy:** `[...array]`, `slice()`, `Array.from()`, `concat()`.
* **Deep copy (nested):** `JSON.parse(JSON.stringify(array))`.

| **Method**               | **Example**                                           | **Type of Copy** | **Notes / Pros & Cons**                                                       |
| ------------------------ | ----------------------------------------------------- | ---------------- | ----------------------------------------------------------------------------- |
| **Spread Operator**      | `const newArray = [...array];`                        | Shallow          | Simple, readable, ES6+ only.                                                  |

### ‚úÖ **Key Takeaways**

1. **Shallow copy** ‚Üí top-level elements copied; nested objects/arrays **still share reference**.
2. **Deep copy** ‚Üí everything copied recursively; original array/object is completely independent.
3. Choose method depending on **browser support, readability, and nesting**.
